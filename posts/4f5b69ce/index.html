零基础Hexo+Github搭建静态个人博客

前言：相信很多人都有写博客的习惯，但是肯定有一部分人和我一样，想拥有一个自己的独立博客。这样不用受个大平台的约束，拥有属于自己风格的独特样式博客。但是对于小白来说，搭建属于自己的独立博客似乎是很难的，往往信心满满的开始动手时，遇到一些自己不懂的专业知识便被吓退了回去。这篇文章从小白的角度出发，让小白也能一步一步将属于自己的独立博客搭建起来，期间也许会遇到问题，但是只要有一个学习的心，这些问题都不再是问题。下面开始进入主题：

搭建过程大概需要这么几个步骤：

​ 1.搭建环境的准备工作（主要是node.js和git环境，以及GitHub账户的设置）

​ 2.安装并初始化Hexo

​ 3.将Hexo和GitHub Page联系起来

​ 4.如何发布新文章

​ 5.如何解析域名到自己的博客

​ 6.Hexo插件的添加

搭建环境的准备工作

主要以下三步：

(1) node.js安装

(2) git安装

(3)GitHub账户设置

1.node.js安装

有些小伙伴可能听到node.js就有点惧怕了，不过在这里不用担心，在这里我们只是用到了node.js的npm包管理工具，在新版的node.js中，已经将npm集成了，所以不必在单独安装npm了。不了解npm的小伙伴可以百度查阅一下.

node.js下载地址：node.js

node.js的安装过程就和安装QQ一样简单，一直点击下一步就行了。

检查node.js是否安装成功：win+R键，然后输入cmd弹出命令框，在命令行内输入node -v,若出现node的版本号则安装成功，可以进行下一步了。

2.安装git

Git是现在世界上最先进的分布式版本控制系统，在这里就不多介绍了，感兴趣的小伙伴可以到

廖雪峰博客学习，他的教程很棒。

git下载地址：[Git](https://git-scm.com/）

git的安装也和QQ一样简单，一路下一步就行了。

检查git是否安装成功：在你的电脑随意位置右击，选中Git bash ,弹出git命令框，在命令行中输入git --version，若弹出版本号则表示你已经安装成功，你可以进入下一步了。

3.github账户设置

首页到GitHub注册一个账号：

注册好后会到回到这样一个界面，如果不一样你试着到处按一下，总会找到的：

然后选择New respository，这个相当于新建了一个仓库，用来存放你的代码之类的：

记住：你取的名字一定要和你的用户名一样，格式就像这样：[用户名].github.io,然后点击create respository,接下来你就可以查看自己的仓库了:

然后点击settings，记住是当前仓库的settings，不是头像那里的settings，进入设置后下拉找到GitHub page，接下来就是开启GitHub-page功能：

点击choose a theme，进入选择一个主题，然后点击select theme

这里我因为创建过了，所以界面有可能会不一样，第一次时界面可能会是这样的，点击Automatic page generator就行了，GitHub会给你创建一个page界面：

上面步骤都做完后，你就可以在线预览[用户名].github.io这个网址了，若能够正常访问，则代表你这一步已经结束了，可以进入下一步了。

题外话：走到这一步是不是感觉很简单呢，心里是不是有点小兴奋，那么继续跟着步骤来，你的博客马上就要建好了。

4.安装Hexo

hexo是什么：Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。这是官方的解释，也很简单明了，总之就是一个博客框架。

6.接下来就是将hexo与GitHub pages联系起来了

第一步：将本地git项目与远程的GitHub建立联系，这时候我们就要配置SSH keys(不明白也没关系，按照步走就行)

首先检查你电脑上的ssh密钥，在电脑任意位置右击，打开git bash,在命令行中输入：cd ~/.ssh（检查本机密钥）如果提示：No such file or directory 说明你是第一次使用git。

如果是第一次使用，则在命令行中输入：ssh-keygen -t rsa -C "邮件地址@youremail.com"
【注】：邮箱地址输入你自己的邮箱地址，其次C是大写。

然后系统会让你输入密码，这个密码用来提交项目的。由于这里我已经使用过了，所以没法演示了，跟着命令行的提示往下输入就行。

【注】输入密码可能会没有*的字样，显示为空白，不用担心，继续输入就行。

最后你会看到the key…之类的，说明配置成功。

第二步：将SSH key 添加到GitHub

打开本地C:\Users\用户名，你会找到.ssh文件夹，如果没找到你就可以试试其他文件夹，理论上默认此文件夹：文件夹里面你会看到如下文件：

如果没有可能被隐藏了，这时你需要设置显示隐藏文件，然后打开id_ras.pub文件，复制里面的内容。然后登陆GitHub，进入settings界面（这里是总的设置界面，不是仓库设置界面），然后点击new key,将刚刚复制的内容分粘贴进去，保存就ok了。

配置完成后就该检查是否成功没有了，在刚刚的命令行窗口中输入：ssh -T git@github.com，

若出现上图界面，则显示成功了，第一册可能会有一个叫你输入Yes/No的选项，输入Y就好了，然后可以继续下一步了。

第三步：现在你已经通过SSH链接到GitHub了，现在配置一下git的相关信息（如果设置过则可以跳过此步骤），在你的命令行中输入：

$ git config --global user.name "aierui"//用户名

$ git config --global user.email "imland@outlook.com"//填写自己的邮箱

到这里就配置完成了，继续下面的步骤了。

7.发布一篇文章

在博客文件夹中右击打开git bash，在命令行中输入：hexo new post "article title"，回车，然后你会在source\ _posts文件夹下看到article title.md 文件，用markdown编辑器就可以编辑的文章了，如果对markdown编辑器不太了解的，可以百度，很简单的。

注意：在生成和部署文章之前需要安装一个扩展，在当前命令行中输入：npm install hexo-deployer-git --save
